Chapter 01 · 这是什么

我们为什么叫里海

里海有很多头衔——最大、最深、最古老。但真正让我们非它不可的，只有一个：独立。它是一座天生的金库。

其一 · 独立

一片不依赖任何人的海

没有出海口 · 不与大洋连通 · 不受潮汐支配

里海不与任何大洋相通，也没有一个流向外海的出口。它不理会全球海平面，不受潮汐与洋流摆布——一个不依赖外部、自成一体的完整系统。这正是比特币「自托管」的镜像：不经过银行、不依赖第三方清算。安全不靠藏，靠的是不接任何一根管道。

其二 · 恒久

被时间原样封存的海

被大地围合 · 存得住，也存得久

它被地质运动彻底围合，原样保存至今。一片海能被这样安静地存放，靠的不是运气，是结构。对一家以长期保管为业的公司，我们没能找到比它更好的图腾。

独立、恒久——所以 Caspian 只做一件事：
安全地存储，守护你手里 BTC 的价值。

我们要服务的，是这样一群人：看得懂比特币的价值，却无法、或不愿亲自持有它——受身份与渠道所限难以合法直接买入，或不想亲手打理交易、私钥与冷存储。
你负责看懂；安全拥有这件事，交给里海。

其三 · 「不可超发」不是承诺，是代码
// bitcoin-core / src/validation.cpp —— 全网每个节点都在运行的共识规则
CAmount GetBlockSubsidy(int nHeight, const Consensus::Params& params)
{
    int halvings = nHeight / params.nSubsidyHalvingInterval; // 每 210,000 块一次
    // 64 次减半后，奖励归零——发行永久停止
    if (halvings >= 64)
        return 0;
    CAmount nSubsidy = 50 * COIN;   // 起始：每块 50 枚
    nSubsidy >>= halvings;            // 每减半一次，右移一位（÷2）
    return nSubsidy;
}

每 21 万个区块，新币奖励减半（nSubsidy >>= halvings 就是一次二进制右移）；64 次之后，函数返回 0，再也发不出一聪。这段代码由全球上万个独立节点各自运行——谁若打包的区块多发一分钱，其余节点即刻判定无效、拒绝接力。2,100 万不是一句口号，是一行所有人都在执行、且改不动的代码。（2017 年，业界最大的算力与公司联盟仅想改一个参数，被普通用户的节点集体否决。）

Chapter 03 · 我们做什么

替你把「拥有」这件事做对

你买的，是一张比特币的长期看涨期权。

不用注册交易所，不用保管私钥，不用学任何技术——像持有一只股票一样，安全地拥有比特币的长期敞口。买入、机构级冷藏、审计、写进可继承的法律结构——这些最容易出事的环节，全部由公司替你扛。你持有的，是这家公司的股份。

托管

机构级合格托管 + 多签冷储。私钥保管这件最容易出事的事，交给制度，而不是交给记性。

传承

股份写得进遗嘱、信托与家族架构——传下去的是一份产权，不是一串没人打得开的密码。

合规

面向美国及国际合格投资人，一条路径清晰、文件齐全、可审计的合法持有通道。

税务

公司「持有不卖」，把应税时点合法后推（递延，而非免税）；持股的税务路径，比自己持币清晰得多。
