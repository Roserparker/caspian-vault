第一章 · 这是什么

我们为什么叫里海

里海有很多头衔——最大、最深、最古老。但真正让我们非它不可的，只有一个：独立。它是一座天生的金库。

其一 · 独立

一片不依赖任何人的海

没有出海口 · 不与大洋连通 · 不受潮汐支配

不与任何大洋相通，不受潮汐摆布——不依赖外部，自成一体。这正是比特币「自托管」的镜像。安全不靠藏，靠的是不接任何一根管道。

细读

它不理会全球海平面，也没有一个流向外海的出口。对应到比特币：不经过银行、不依赖任何第三方清算——系统的完整性来自结构本身。

其二 · 恒久

被时间原样封存的海

被大地围合 · 存得住，也存得久

被地质运动彻底围合，原样保存至今。对一家以长期保管为业的公司，没有比它更好的图腾。

细读

一片海能被这样安静地存放三千万年，靠的不是运气，是结构。存得住，也存得久。

其三 · 「不可超发」不是承诺，是代码
// bitcoin-core / src/validation.cpp —— 全网每个节点都在运行的共识规则
CAmount GetBlockSubsidy(int nHeight, const Consensus::Params& params)
{
    int halvings = nHeight / params.nSubsidyHalvingInterval; // 每 210,000 块一次
    // 64 次减半后，奖励归零——发行永久停止
    if (halvings >= 64)
        return 0;
    CAmount nSubsidy = 50 * COIN;   // 起始：每块 50 枚
    nSubsidy >>= halvings;            // 每减半一次，右移一位（÷2）
    return nSubsidy;
}

2,100 万不是口号，是一行所有人都在执行、且改不动的代码。

细读

每 21 万个区块，新币奖励减半（nSubsidy >>= halvings 是一次二进制右移）；64 次之后函数返回 0，再也发不出一聪。全球上万个独立节点各自运行这段代码——谁多发一分钱，其余节点即刻判定无效。2017 年，业界最大的算力联盟想改一个参数，被普通用户的节点集体否决。

安全怎么做到：三道门

比特币只认钥匙，不认人——被偷无法追回，转走无法撤销。所以安全不是卖点，是产品本身。库房有三道门：

门一 · 断网

钥匙一生不碰网

私钥在一台从未联网的机器上出生、保管、签名。黑客的所有本事都需要一根网线——我们不给。

密钥生成 · 存储 · 签名全程离线（air-gapped 冷储存）
门二 · 多签

一把锁，多把钥匙，分居多地

动币需要分存异地的多把钥匙同时到场——像银行金库的双人开锁。任何单人、单点，都动不了库。

M-of-N 阈值签名（multisig）· 无单点故障

不让任何人管所有钥匙

持牌托管商参与保管：保险、独立审计、监管盯着。没有人能一手遮天。
